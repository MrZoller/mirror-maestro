Manage GitLab mirrors across multiple instance pairs

Welcome to GitLab Mirror Wizard

This tool helps you manage GitLab mirrors across multiple instance pairs. Here's how to get started:

1. Add GitLab Instances

Configure your source and target GitLab instances with their API tokens.

2. Create Instance Pairs

Define pairs of GitLab instances where mirrors will be created.

3. Configure Group Settings

Add group access tokens and (optionally) group-level mirror default overrides.

4. Configure Mirrors

Create and manage mirrors between projects in your instance pairs.

5. Monitor & Maintain

View mirror status, trigger updates, and import/export configurations.

Features

  • Easy mirror creation with minimal configuration
  • Support for multiple GitLab instance pairs
  • Push and pull mirror support
  • Bulk import/export of mirror configurations
  • Real-time status monitoring
  • Force mirror updates on demand

Group Settings

Manage group access tokens (for HTTPS mirroring) and group-level mirror default overrides (that take precedence over pair defaults).

Note: Group access tokens are required for mirrors to authenticate via HTTPS. Create a token for each group that contains projects you want to mirror. The token must have read_repository and write_repository scopes.
