# Security Policy

## Reporting a vulnerability

Please **do not** report security vulnerabilities via public GitHub issues.

Instead, report them privately by:

1. Opening a GitHub issue with **no sensitive details** and asking for a private contact channel, **or**
2. Contacting the maintainer through the contact method listed on the projectâ€™s GitHub profile.

If you're unsure whether something is a security issue, err on the side of reporting it privately.

## What to include

- A description of the issue and impact
- Steps to reproduce (proof of concept if possible)
- Affected versions / commits (if known)
- Any suggested fix or mitigation

## Handling secrets

This project manages GitLab tokens. Please avoid including real tokens in:

- bug reports
- logs
- screenshots
- exported JSON

If you accidentally exposed a token, rotate it immediately.
